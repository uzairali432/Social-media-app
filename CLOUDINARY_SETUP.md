# Cloudinary Setup (Account & Project Configuration)

This document lists the minimal steps to set up Cloudinary for the project. This is configuration only — no code that uploads or uses Cloudinary is added here.

1. Create a Cloudinary account
   - Visit https://cloudinary.com/ and sign up for a free account.

2. Get your credentials
   - In the Cloudinary Console (Dashboard) find:
     - Cloud name
     - API key
     - API secret

3. Store credentials in server environment
   - Add the following environment variables to `server/.env` (or set them in your deployment environment):
     - `CLOUDINARY_CLOUD_NAME` — your cloud name
     - `CLOUDINARY_API_KEY` — your API key
     - `CLOUDINARY_API_SECRET` — your API secret
   - These variables are now generated by `setup.bat` (placeholder values). Update them with real values before using Cloudinary.

4. Install the Cloudinary SDK (local development)
   - The `cloudinary` package is already added to `server/package.json` dependencies. Run:

```powershell
cd server
npm install
```

5. Deployment notes
   - In production, configure environment variables securely (CI/CD secrets, hosting env variables).
   - Do NOT commit secrets to source control.

6. Next steps (implementation, optional)
   - Create a small `server/utils/cloudinary.js` that configures the Cloudinary client using the env vars.
   - Use the SDK in upload endpoints or background jobs.
